HTTP = Hyper Text Transfer Protocol
designed to enable communications between clients and servers.
- Media independent - any type of data can be sent with HTTP as long as sender/reciever can process.
- Stateless - Server/client are aware of each other only when a data transaction is taking place.
------ Neither the client of server retains information.

HTTP is not secure. Clear text usually served on port 80

HTTPS = layering HTTP with SSL or TLS
----- Usually port 443
----- Creates a secure channel over an unsecure network.


HTTP Methods (1.1 has the 3 methods from 1.0 as well.)
  • HTTP 1.0 - GET, POST, HEAD
    • Head - only return headers and status code, but without content.
    • Get - standard method used to retrieve resouce content given a URI
    • Post - used to submit content to server, such as forms, data, files, etc.
  • HTTP 1.1 - OPTIONS, PUT, DELETE, TRACE, PATCH, CONNECT
    • Options - provides communication options for target
    • Put - request to store a resource given by the URI
    • Delete - Remove all representations of the target resource given by the URI
    • Echos the recieved request so that the client can see what changes have been made by intermediaries.
    • Patch - apply partial modificaitons to a resource
    • Connect - establish a tunnel to the server, identified by the given URI (HTTPS)
